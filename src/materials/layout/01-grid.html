The grid system it’s based on a 12 column layout and has multiple tiers, one for each media query range. It can be use it with Sass mixins or the predefined classes.

How it works

  • There are three major components—containers, rows, and columns.
  • Containers—.container for fixed width or .container-fluid for full width—center the site’s contents and help align the grid content.
  • Rows are horizontal groups of columns that ensure that the columns are lined up properly.
  • Content should be placed within columns, and only columns may be immediate children of rows.
  • Column classes indicate the number of columns to use out of the possible 12 per row. So, for a three equal-width columns, it has to use .col-sm-4, for example.
  • Column widths are set in percentages, so they’re always fluid and sized relative to their parent element.
  • Columns have horizontal padding to create the gutters between individual columns.
  • There are five grid tiers, one for each responsive breakpoint: extra small, small, medium, large, and extra large.
  • Grid tiers are based on minimum widths, meaning they apply to that one tier and all those above it (e.g., .col-sm-4 applies to small, medium, large, and extra large devices).
  • It can be use predefined grid classes or Sass mixins for more semantic markup.
